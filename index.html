🔧 Tutorial Mikrotik & Winbox

Panduan Lengkap Konfigurasi Network - Dari Dasar hingga Mahir

🚀 Cara Masuk Winbox Mikrotik

💡 Info: Winbox adalah aplikasi GUI untuk konfigurasi Mikrotik RouterOS. Download dari mikrotik.com/download

Metode 1: MAC Address (Paling Mudah)

1 Buka Winbox
Jalankan winbox.exe atau winbox64.exe (tidak perlu install)
2 Klik tombol "..." (Neighbors)
Winbox akan otomatis scan Mikrotik yang terhubung di jaringan
3 Pilih Mikrotik dari List
Klik pada MAC Address Mikrotik yang muncul (format: XX:XX:XX:XX:XX:XX)
4 Login
Username: admin
Password: (kosongkan jika baru/default)
Klik Connect

Metode 2: IP Address Manual

⚠️ Kapan pakai metode ini? Jika Mikrotik tidak muncul di list Neighbors (masalah network/firewall)
1 Pastikan PC dan Mikrotik Satu Network
Cek IP Mikrotik di label RouterBOARD (biasanya 192.168.88.1)
2 Setting IP PC Anda
Windows:
Control Panel → Network Connections → Klik kanan Ethernet → Properties → IPv4
IP Address:
192.168.88.2
Subnet Mask:
255.255.255.0
Gateway:
192.168.88.1
3 Tambah IP Manual di Winbox
Di kolom "Connect To", ketik manual: 192.168.88.1
Lalu klik Connect

❌ Troubleshooting: Mikrotik Tidak Muncul di List

Problem: Neighbor list kosong / Mikrotik tidak terdeteksi
1 Cek Kabel Network
Pastikan kabel LAN terhubung dari PC ke port Mikrotik (lampu indikator menyala)
2 Disable Firewall/Antivirus Sementara
Windows Firewall atau antivirus bisa block neighbor discovery protocol
3 Reset IP di Network Adapter
Buka CMD (Admin), jalankan:
ipconfig /release
ipconfig /renew
4 Coba IP yang Nyangkut di NCP/DHCP
Kadang IP lama masih nyangkut. Coba setting manual:
• 192.168.88.2 (default Mikrotik network)
• 192.168.1.2 (network umum lain)
• 10.10.10.2 (alternatif lain)

🔗 Konfigurasi: Beda IP Bisa Ping (Tanpa Internet)

🎯 Tujuan: Menghubungkan 2 PC dengan IP berbeda melalui Mikrotik sebagai router, tanpa koneksi internet.

Topologi Network

PC 1
192.168.1.10
MIKROTIK
Router
PC 2
192.168.2.10

Step-by-Step Configuration

1 Login ke Winbox
Ikuti cara login di tab "Winbox Setup"
2 Setting IP Address untuk Interface 1
Menu: IP → Addresses → Klik tombol [+]
Address:
192.168.1.1/24
Interface:
ether1
Klik OK
3 Setting IP Address untuk Interface 2
Klik [+] lagi untuk tambah IP kedua:
Address:
192.168.2.1/24
Interface:
ether2
Klik OK
✅ Penjelasan:
• 192.168.1.1/24 = IP untuk network pertama (subnet 192.168.1.0)
• 192.168.2.1/24 = IP untuk network kedua (subnet 192.168.2.0)
• /24 = Subnet mask 255.255.255.0 (255 host per network)
4 Setting IP di PC 1
Hubungkan PC 1 ke ether1 Mikrotik
Setting IP manual di PC 1:
IP Address:
192.168.1.10
Subnet Mask:
255.255.255.0
Gateway:
192.168.1.1
5 Setting IP di PC 2
Hubungkan PC 2 ke ether2 Mikrotik
Setting IP manual di PC 2:
IP Address:
192.168.2.10
Subnet Mask:
255.255.255.0
Gateway:
192.168.2.1
6 Test Koneksi dengan Ping
Dari PC 1, buka CMD dan ping ke PC 2:
ping 192.168.2.10
Dari PC 2, buka CMD dan ping ke PC 1:
ping 192.168.1.10
🎉 Berhasil! Jika muncul "Reply from..." berarti kedua PC sudah bisa berkomunikasi meskipun beda subnet/network!

🌐 Konfigurasi: Beda IP Bisa Ping + Internet

🎯 Tujuan: Sama seperti sebelumnya, tapi sekarang semua PC juga bisa akses internet melalui Mikrotik!

Topologi Network dengan Internet

INTERNET
ISP/Modem
MIKROTIK
Router + NAT
PC 1
192.168.1.10
PC 2
192.168.2.10

Step-by-Step Configuration

1 Lakukan Konfigurasi Dasar (Langkah 1-5 dari Tab Sebelumnya)
Setup IP untuk ether1, ether2, PC 1, dan PC 2 seperti panduan "Tanpa Internet"
2 Hubungkan Mikrotik ke Internet
Sambungkan kabel dari Modem/ISP ke port ether3 (atau port WAN) di Mikrotik
3 Setting DHCP Client untuk WAN
Menu: IP → DHCP Client → Klik [+]
Interface:
ether3
☑️ Use Peer DNS:
Centang (enabled)
☑️ Add Default Route:
Centang (enabled)
Klik OK
💡 Penjelasan: DHCP Client membuat Mikrotik mendapat IP otomatis dari modem/ISP Anda (seperti PC biasa). Status "bound" = koneksi berhasil!
4 Setting DNS Server
Menu: IP → DNS
Servers:
8.8.8.8, 8.8.4.4
☑️ Allow Remote Requests:
Centang (enabled)
Klik OK
5 Setup NAT (Masquerade) PENTING!
Menu: IP → Firewall → Tab NAT → Klik [+]

Tab General:
Chain:
srcnat
Out. Interface:
ether3
Tab Action:
Action:
masquerade
Klik OK
⚠️ Kenapa Butuh NAT?
NAT (Network Address Translation) mengubah IP private (192.168.x.x) menjadi IP public yang diberikan ISP. Tanpa NAT, PC Anda tidak bisa akses internet!
6 Setting DNS di PC 1 dan PC 2
Tambahkan DNS Server di setting IP masing-masing PC:

PC 1:
Preferred DNS:
8.8.8.8
Alternate DNS:
8.8.4.4
PC 2: (sama seperti PC 1)
7 Test Koneksi Lengkap
Test 1 - Ping antar PC (harus berhasil):
ping 192.168.2.10 (dari PC 1 ke PC 2)
ping 192.168.1.10 (dari PC 2 ke PC 1)
Test 2 - Ping Gateway (harus berhasil):
ping 192.168.1.1 (dari PC 1)
ping 192.168.2.1 (dari PC 2)
Test 3 - Ping Internet (harus berhasil):
ping 8.8.8.8 (Google DNS)
ping google.com
🎉 Sempurna! Sekarang:
✅ PC 1 dan PC 2 bisa saling ping (beda network)
✅ Kedua PC bisa akses internet
✅ Mikrotik berfungsi sebagai router + gateway

🔧 Troubleshooting & Tips

❌ Problem 1: Ping Tidak Berhasil Antar PC

Symptom: Request timed out / Destination host unreachable
1 Cek IP Configuration
Di CMD, jalankan: ipconfig
Pastikan IP, Subnet, dan Gateway sesuai konfigurasi
2 Cek IP di Mikrotik
IP → Addresses
Pastikan ada 2 IP (192.168.1.1/24 dan 192.168.2.1/24) dengan interface yang benar
3 Disable Windows Firewall Sementara
Control Panel → Windows Defender Firewall → Turn off (untuk testing)
4 Ping ke Gateway Dulu
Dari PC 1: ping 192.168.1.1
Dari PC 2: ping 192.168.2.1
Jika ini berhasil tapi antar-PC gagal, masalah ada di routing

❌ Problem 2: Tidak Bisa Akses Internet

Symptom: Bisa ping antar PC, tapi tidak bisa browsing/ping 8.8.8.8
1 Cek Status DHCP Client
IP → DHCP Client
Status harus "bound" (warna hijau)
Jika bukan, masalah di koneksi modem/ISP
2 Cek Firewall NAT
IP → Firewall → NAT
Harus ada 1 rule: srcnat + masquerade + out-interface ether3
Jika tidak ada, buat rule baru!
3 Cek DNS Configuration
IP → DNS
Servers: 8.8.8.8, 8.8.4.4
Allow Remote Requests: ✅ Centang
4 Test dari Mikrotik Langsung
Tools → Ping
Ping: 8.8.8.8
Jika berhasil dari Mikrotik tapi gagal dari PC → masalah di NAT/DNS

❌ Problem 3: DHCP Client Tidak Bound

Symptom: DHCP Client status "searching" terus-menerus
1 Cek Kabel WAN
Pastikan kabel dari modem ke ether3 tersambung dengan baik
2 Cek Modem Status
Pastikan modem sudah online (lampu indikator internet menyala)
3 Ganti Interface WAN
Coba pindah ke port lain (ether4/ether5), lalu update DHCP Client

💡 Tips & Best Practices

✅ Selalu Backup Konfigurasi!
Menu: Files → Backup → Simpan file .backup
🔒 Keamanan:
• Ganti password default Mikrotik
• Disable unused services (Telnet, FTP, etc.)
• Enable firewall filter rules untuk proteksi
📝 Dokumentasi:
Catat semua IP address dan konfigurasi Anda untuk referensi di masa depan!

🎓 Konsep Penting yang Perlu Dipahami

Subnet & CIDR Notation:
/24:
255.255.255.0 (256 host, 254 usable)
/16:
255.255.0.0 (65536 host)
/30:
255.255.255.252 (4 host, 2 usable - untuk link point-to-point)
Private IP Ranges (RFC 1918):
Class A:
10.0.0.0 - 10.255.255.255
Class B:
172.16.0.0 - 172.31.255.255
Class C:
192.168.0.0 - 192.168.255.255
